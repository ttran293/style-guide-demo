Data Summary Snippets

Data summary snippets turn measurements, quality checks, and workflow counts into compact decision support. Use them when users need the state of a dataset before choosing the next action.

Summary patterns

Pattern Best for Rule
Metric strip Three to four high-level values Keep labels short and values directly comparable
Insight card One finding with supporting details Explain what changed and why it matters
Review summary Before export, approval, or batch actions Pair status with the recommended next step

Guidelines

  • Use summaries to support a decision, not to replace detailed tables or charts.
  • Keep values visible, but use labels and helper text to prevent misinterpretation.
  • Put warnings close to the action they affect.
  • Use trend color sparingly; neutral text is preferred unless the direction changes user behavior.

Accessibility

  • Do not encode status with color alone; include text such as "warnings" or "ready".
  • Use headings for each metric group so screen reader users can scan summaries.
  • Keep action order consistent: secondary actions first, primary action last.
