Page Header Snippets

Page header snippets combine title, status, metadata, and primary actions into a reusable top-of-page pattern. Use them to make complex workflows feel oriented before users reach the main content.

Choose the right header

Snippet Use for Main behavior
Workspace header Project, dataset, or acquisition pages Shows current status, ownership, and the primary workflow action
Filter header Search-heavy lists and review queues Pairs a short title with persistent filter controls
Review header Approval, QA, or inspection screens Surfaces risk, progress, and next action without opening a panel

When to use

  • Use on pages where users need context before acting.
  • Use when status, metadata, and actions belong to the entire page.
  • Use one primary action at most; keep secondary actions visually quieter.

When not to use

  • Do not use for small cards or modal sections; use a card header or dialog title instead.
  • Do not repeat the same metadata again in the first panel unless it changes state.
  • Do not put destructive actions in the header without confirmation.

Accessibility

  • Keep the page title as the first meaningful heading in the content area.
  • Use real buttons for actions and real form controls for filters.
  • Badges supplement the heading; they should not be the only place status is explained.
