Button

A button is a UI control that triggers an instantaneous action. It is distinct from state buttons, which toggle persistent state. Allow one primary action per panel; primary labels should be verbs.

Style variants

Variant Usage
Primary Main panel action — Launch, Apply, Add; may close panel or start long processing
Secondary Subsection actions; shorter height; may open settings dialogs
Tertiary Adjacent to a related control — Browse, Reset

Icon button

Icon-only buttons save space for familiar actions (Save, Add, Settings). Always provide an accessible name via aria-label or visually hidden text.

  • Use 16px Font Awesome icons inside 28px primary / 24px secondary hit targets.
  • Pair icon + text when the action is ambiguous or first-time.

Link button

Link buttons look like hyperlinks but use <button> semantics for in-app actions that do not navigate. For true navigation, use Hyperlinks.

Link icon button

Combines link appearance with a leading icon — common for help topics and external docs opened in context.

Split button

A split button pairs a primary action with a menu of related alternatives. The main area runs the default action; the chevron opens a dropdown.

  • Default action must match the most common user choice.
  • Menu items use the same labels as a standard Dropdown action menu.

Button group

Visually joins related options so users pick one of a small set (view mode, alignment, sort order).

  • Use for 2–5 mutually exclusive options; more than five — use a dropdown.
  • Mark the selected option with aria-pressed="true" or .is-active styling.
  • Shared outer corners only — inner borders collapse between segments.

Appearance

  • Button height ≈ 2% of vertical monitor resolution (~20–35px at 1080p, ~40–60px at 4K).
  • Primary: brand blue background, white text; disabled uses neutral gray background.
  • Secondary/tertiary: neutral background, dark text; disabled text muted.
  • Include text when space permits; icon-only for standard actions (Browse, Save) when space is limited.
  • Icons + text when the button may collapse to icon-only or the icon needs definition.

Usage guidelines

  • Only one primary button per panel.
  • Primary button text should be a verb; prefer text-only on primary buttons.
  • Secondary buttons align with other controls in a subsection.
  • Tertiary buttons sit beside their related control (e.g. filename + Browse).
  • Help links use hyperlink appearance, not button styling.
