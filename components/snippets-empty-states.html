Empty State Snippets

Empty states explain why content is missing and give users a practical next step. They should feel calm, specific, and recoverable rather than decorative.

Empty state types

Type Message goal Action style
First use Invite the user to create or import content One primary action
No results Explain that filters are hiding content Secondary reset action
Blocked Tell the user what dependency is missing Specific fix or request access action

Example - first use

No images in this workspace yet

Import a TIFF, JPEG, or OME file to begin building a review set.

Example - no results

No matching files

Try a shorter filename, remove confidence filters, or search by acquisition date.

Example - blocked state

Protocol library unavailable

You can review existing results, but adding a new protocol requires access to the shared library.

Guidelines

  • Lead with the reason content is missing, not a generic "nothing here" message.
  • Use one preferred action when the user can recover immediately.
  • Use secondary or tertiary actions when the next step is optional.
  • Keep illustrations small and symbolic; content and actions carry the meaning.

Accessibility

  • Do not rely on the icon alone; the heading and body copy must explain the state.
  • Use semantic buttons for recovery actions.
  • If an empty state appears after filtering, announce the result count in the surrounding app logic.
