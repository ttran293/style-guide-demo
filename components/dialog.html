Dialogs

A dialog is a modal UI surface that focuses user attention on a task or decision. Titles use sentence case; allow one primary action per dialog.

Modal vs modeless

Type Behavior Use when
Modal Blocks rest of app until dismissed Confirmations, irreversible actions, critical info that must be addressed
Modeless User can interact elsewhere Tools or settings that adjust another active feature with immediate visible effect

Use modals sparingly — they are interruptive. Low/medium priority info belongs in a snackbar or toast instead.

Anatomy

Element Description
Container Raised surface above other UI
Title Brief, clear statement — avoid “Are you sure?” or alarmist copy
Body Concise context and consequences; do not repeat the title
Divider Optional separator between header and body
Actions Max two actions recommended; one primary only

Button placement

Primary action on the right, secondary on the left (Z-pattern reading). User reviews cancel/dismiss before committing to the primary action. Labels should be outcome verbs: Delete, Save, Continue — not vague Yes/Submit.

  • Single action: acknowledgment only (OK, Got it, Close)
  • Two actions: confirming (primary) + dismissing (secondary)

Guidelines

  • Primary button is a verb — Apply, Save, Delete.
  • Escape dismisses modal overlays; trap focus inside the dialog.
  • Use low-contrast divider between header and body, not full enclosing borders.
