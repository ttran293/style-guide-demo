Accessibility

Accessibility guidelines ensure products can be used by people with diverse abilities. Follow WCAG 2.1 AA standards.

WCAG 2.1 AA — four principles

Principle Requirement
Perceivable Information presentable in ways users can perceive
Operable Interface components operable by all users
Understandable Information and operation must be understandable
Robust Content works with assistive technologies

Contrast requirements

Element Minimum ratio
Normal text 4.5:1
Large text (18pt or 14pt bold) 3:1
UI components (borders, focus indicators) 3:1

Key practices

  • All interactive elements reachable via keyboard; logical tab order.
  • Use semantic HTML (<button>, <nav>, proper headings).
  • Do not rely on color alone — pair with icons or text.
  • Provide ARIA labels where visible text is insufficient.
  • Honor prefers-reduced-motion and high contrast settings.
